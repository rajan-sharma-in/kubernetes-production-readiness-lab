# Kubernetes Production Readiness Lab

Kubernetes Production Readiness Lab is a hands-on repository for mentoring and learning how Kubernetes workloads move from basic deployments to production-ready systems.

This repository is structured for contributors exploring:

- Kubernetes workload design and validation
- Helm chart layout and release practices
- Terraform-based infrastructure examples
- CI/CD automation
- Bash-based operational checks
- Troubleshooting and production-readiness reviews

## Repository Structure

```text
.
├── docs/
├── examples/
├── scripts/
└── .github/
```

## Goals

- Build practical Kubernetes examples
- Improve deployment reliability and maintainability
- Document operational debugging workflows
- Create reusable validation and health-check scripts
- Support LFX mentorship-style contribution and review workflows

## Getting Started

1. Review the files in `docs/`.
2. Explore the examples under `examples/`.
3. Run the helper scripts in `scripts/` against a Kubernetes cluster.
4. Follow `CONTRIBUTING.md` before opening changes.

## Mentorship Scope

This lab focuses on production-readiness thinking for cloud-native systems, including:

- security
- observability
- automation
- deployment validation
- troubleshooting
- operational discipline
