# Contributing

Thank you for contributing to Kubernetes Production Readiness Lab.

## Contribution Model

This repository is intended to be public and contribution-friendly, while keeping protected branches restricted to maintainers.

- Do not push directly to `main`.
- Create a feature branch for each change.
- Open a pull request for review before merge.
- Expect CI checks and maintainer review before approval.

## How To Contribute

- Improve documentation in `docs/`
- Add or refine examples in `examples/`
- Extend operational scripts in `scripts/`
- Fix issues related to Kubernetes validation, troubleshooting, and readiness checks

## Development Guidelines

- Keep changes focused and easy to review.
- Prefer clear documentation over implicit behavior.
- Test scripts before submitting changes.
- Use descriptive commit messages.
- Avoid introducing breaking changes without documenting them.

## Pull Request Expectations

- Explain the purpose of the change.
- Describe how the change was validated.
- Include sample commands or expected output when relevant.
- Update documentation if behavior or structure changes.
- Keep pull requests focused and scoped to one logical change.
- Do not include secrets, kubeconfigs, tokens, or private infrastructure details.

## Review Standard

Contributions should improve reliability, clarity, security, maintainability, or learning value.
